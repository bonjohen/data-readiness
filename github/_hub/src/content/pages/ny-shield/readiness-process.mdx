<div class="section">
  <h2>Phase Details</h2>

  <div class="phase-detail">
    <h3>1. Intake &amp; Scoping <span class="phase-duration">Week 1</span></h3>
    <p>We start by understanding your data practices, existing safeguards, and service provider relationships.</p>
    <ul>
      <li><strong>NY resident data inventory</strong> &mdash; identify all private information of New York residents your organization owns or licenses</li>
      <li><strong>Current safeguards documentation</strong> &mdash; assess existing administrative, technical, and physical safeguards already in place</li>
      <li><strong>Service provider mapping</strong> &mdash; catalog all vendors and service providers with access to private information</li>
      <li><strong>Breach notification procedure review</strong> &mdash; evaluate existing incident response and breach notification capabilities</li>
    </ul>
  </div>

  <div class="phase-detail">
    <h3>2. Assessment <span class="phase-duration">Week 2&ndash;3</span></h3>
    <p>We evaluate your current safeguards posture against SHIELD Act requirements.</p>
    <ul>
      <li><strong>Administrative safeguards evaluation</strong> &mdash; review coordinator designation, risk identification, employee training, and service provider contracts</li>
      <li><strong>Technical safeguards assessment</strong> &mdash; assess network/software risks, data processing protections, attack detection, and control testing</li>
      <li><strong>Physical safeguards review</strong> &mdash; evaluate information storage risks, intrusion detection, unauthorized access protection, and data disposal</li>
      <li><strong>Small business safe harbor determination</strong> &mdash; assess whether your organization qualifies for the scaled-down security program</li>
    </ul>
  </div>

  <div class="phase-detail">
    <h3>3. Outputs <span class="phase-duration">Week 3&ndash;4</span></h3>
    <p>We deliver the artifacts that define your path to SHIELD Act compliance.</p>
    <ul>
      <li><strong>Data security program documentation</strong> &mdash; comprehensive documentation of your administrative, technical, and physical safeguards</li>
      <li><strong>Risk assessment report</strong> &mdash; documented assessment of internal and external risks to the security of private information</li>
      <li><strong>Service provider contract requirements</strong> &mdash; template language and checklist for vendor agreements requiring appropriate safeguards</li>
      <li><strong>Breach notification procedure templates</strong> &mdash; step-by-step procedures for breach detection, assessment, and AG/DFS/DOCS notification</li>
    </ul>
  </div>

  <div class="phase-detail">
    <h3>4. Follow-on <span class="phase-duration">Ongoing</span></h3>
    <p>After the readiness sprint, maintaining compliance requires ongoing attention to your data security program.</p>
    <ul>
      <li><strong>Annual program updates</strong> &mdash; adjust the data security program in light of business changes, new threats, and evolving circumstances</li>
      <li><strong>Employee training program</strong> &mdash; regular training on data security practices and procedures for all employees</li>
      <li><strong>Service provider monitoring</strong> &mdash; ongoing assessment of service provider safeguard compliance and contract adherence</li>
    </ul>
  </div>
</div>
