<div class="section">
  <h2>Phase Details</h2>

  <div class="phase-detail">
    <h3>1. Intake &amp; Scoping <span class="phase-duration">Week 1</span></h3>
    <p>We start by understanding your payment processing environment and current compliance posture.</p>
    <ul>
      <li><strong>CDE scoping and data flow analysis</strong> &mdash; identify all systems, networks, and processes that store, process, or transmit cardholder data</li>
      <li><strong>SAQ vs. ROC determination</strong> &mdash; determine the appropriate validation type based on merchant level and transaction volume</li>
      <li><strong>Network segmentation review</strong> &mdash; evaluate existing segmentation to define CDE boundaries</li>
      <li><strong>Current control inventory</strong> &mdash; document existing security controls and compliance documentation</li>
    </ul>
  </div>

  <div class="phase-detail">
    <h3>2. Assessment <span class="phase-duration">Week 2&ndash;3</span></h3>
    <p>We evaluate your current control posture against all 12 PCI DSS requirements.</p>
    <ul>
      <li><strong>12-requirement gap analysis</strong> &mdash; systematic review of controls across all requirements and sub-requirements</li>
      <li><strong>Network architecture review</strong> &mdash; validate CDE isolation, firewall rules, and segmentation effectiveness</li>
      <li><strong>Access control and authentication audit</strong> &mdash; evaluate user identification, MFA implementation, and access restriction controls</li>
      <li><strong>Vulnerability management evaluation</strong> &mdash; assess patching cadence, ASV scan readiness, and penetration testing scope</li>
    </ul>
  </div>

  <div class="phase-detail">
    <h3>3. Outputs <span class="phase-duration">Week 3&ndash;4</span></h3>
    <p>We deliver the artifacts that define your path to PCI DSS compliance.</p>
    <ul>
      <li><strong>Scoping document with CDE boundaries</strong> &mdash; documented cardholder data environment with system inventory and network diagrams</li>
      <li><strong>Gap register mapped to 12 requirements</strong> &mdash; every requirement mapped to current state, with gaps ranked by risk and remediation effort</li>
      <li><strong>SAQ/ROC preparation roadmap</strong> &mdash; pre-populated assessment template or evidence mapping for QSA fieldwork</li>
      <li><strong>Remediation prioritization matrix</strong> &mdash; prioritized plan to close gaps with owners, timelines, and resource requirements</li>
    </ul>
  </div>

  <div class="phase-detail">
    <h3>4. Follow-on <span class="phase-duration">Ongoing</span></h3>
    <p>After the readiness sprint, maintaining compliance requires ongoing monitoring and validation.</p>
    <ul>
      <li><strong>ASV scan program</strong> &mdash; coordinate quarterly external vulnerability scans with an Approved Scanning Vendor</li>
      <li><strong>Penetration testing schedule</strong> &mdash; annual penetration testing coordination with scope aligned to CDE boundaries</li>
      <li><strong>Annual reassessment preparation</strong> &mdash; support annual PCI DSS revalidation and evidence collection</li>
    </ul>
  </div>
</div>
