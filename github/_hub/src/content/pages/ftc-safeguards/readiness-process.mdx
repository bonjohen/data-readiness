<div class="section">
  <h2>Phase Details</h2>

  <div class="phase-detail">
    <h3>1. Intake &amp; Scoping <span class="phase-duration">Week 1</span></h3>
    <p>We start by understanding your institution, systems, and current security posture.</p>
    <ul>
      <li><strong>FTC jurisdiction confirmation</strong> &mdash; verify that your institution falls under the Safeguards Rule's scope</li>
      <li><strong>Current security program review</strong> &mdash; assess existing policies, procedures, and technical controls</li>
      <li><strong>Qualified individual identification</strong> &mdash; evaluate current designation or identify candidates for the qualified individual role</li>
      <li><strong>Customer information inventory</strong> &mdash; catalog all systems, applications, and processes that handle customer information</li>
    </ul>
  </div>

  <div class="phase-detail">
    <h3>2. Assessment <span class="phase-duration">Week 2&ndash;3</span></h3>
    <p>We evaluate your current security posture against the amended Safeguards Rule requirements.</p>
    <ul>
      <li><strong>Risk assessment methodology review</strong> &mdash; evaluate whether your risk assessment meets the rule's written assessment requirements</li>
      <li><strong>Technical controls evaluation</strong> &mdash; assess encryption, secure development, change management, and monitoring capabilities</li>
      <li><strong>Access control and MFA audit</strong> &mdash; review authentication mechanisms and access controls for customer information systems</li>
      <li><strong>Incident response plan review</strong> &mdash; evaluate response procedures, roles, and communication protocols against rule requirements</li>
    </ul>
  </div>

  <div class="phase-detail">
    <h3>3. Outputs <span class="phase-duration">Week 3&ndash;4</span></h3>
    <p>We deliver the artifacts that define your path to Safeguards Rule compliance.</p>
    <ul>
      <li><strong>Written information security program</strong> &mdash; comprehensive WISP document meeting all rule requirements</li>
      <li><strong>Risk assessment documentation</strong> &mdash; written risk assessment with identified threats, evaluation criteria, and mitigation plans</li>
      <li><strong>Technical controls gap register</strong> &mdash; every control gap ranked by risk with owners and remediation timelines</li>
      <li><strong>Board reporting framework</strong> &mdash; template and process for the qualified individual's annual report to the board</li>
    </ul>
  </div>

  <div class="phase-detail">
    <h3>4. Follow-on <span class="phase-duration">Ongoing</span></h3>
    <p>After the readiness sprint, continued support ensures sustained compliance.</p>
    <ul>
      <li><strong>Annual risk assessment updates</strong> &mdash; periodic reassessment as threats, systems, and business operations evolve</li>
      <li><strong>Penetration testing program</strong> &mdash; design and oversight of annual penetration testing or continuous monitoring with vulnerability assessments</li>
      <li><strong>Continuous monitoring implementation</strong> &mdash; ongoing security program monitoring and board reporting support</li>
    </ul>
  </div>
</div>
